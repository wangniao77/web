# 数字建模系统 · 前后端接口文档

> 版本：2026-07-07  
> 后端：FastAPI + Tortoise ORM  
> 前端：Vue 3 + Vite + Axios  
> API 前缀：`/api/v1`（健康检查除外）

---

## 1. 概述

本系统提供三类大屏驾驶舱与一套管理后台接口：

| 模块 | 路由前缀 | 鉴权 | 说明 |
|------|----------|------|------|
| 学院大屏 | `/api/v1/college` | 否 | 学院发展与治理驾驶舱，前端主对接模块 |
| 学校大屏 | `/api/v1/university` | 否 | 全校综合驾驶舱 |
| 学生大屏 | `/api/v1/student` | 否 | 个人学业成长驾驶舱 |
| 认证 | `/api/v1/auth` | 部分 | 登录 / 当前用户 |
| 超级管理员 | `/api/v1/admin` | 是（superadmin） | 通用 CRUD |
| 健康检查 | `/api/health` | 否 | 服务存活探测 |

**Swagger 文档**：后端启动后访问 `http://localhost:8000/docs`

## 3. 通用约定

### 3.1 统一响应结构

除 HTTP 层错误外，业务接口均返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": { },
  "timestamp": 1720000000000
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | `number` | 业务状态码，`0` 表示成功 |
| `message` | `string` | 提示信息 |
| `data` | `T \| null` | 业务数据 |
| `timestamp` | `number` | 毫秒时间戳 |

**前端取值方式**：

```typescript
const res = await client.get<ApiResponse<OverviewHubDTO>>('/college/overview/hub', { params })
const payload = unwrapApiData(res)  // 即 res.data.data，并校验 code === 0
```

对应文件：`frontend/src/api/unwrap.ts`、`frontend/src/types/common.ts`

### 3.2 HTTP 错误

| HTTP 状态码 | 场景 | 响应体示例 |
|-------------|------|------------|
| 401 | 未登录 / Token 无效 | `{"detail": "未提供认证令牌"}` |
| 403 | 权限不足 | `{"detail": "权限不足"}` |
| 404 | 资源不存在 | `{"detail": "学生不存在"}` |
| 400 | 参数或业务校验失败 | `{"detail": "创建账号必须提供 password"}` |

### 3.3 鉴权（JWT）

需要鉴权的接口在请求头携带：

```
Authorization: Bearer <token>
```

Token 由 `POST /api/v1/auth/login` 获取。前端在 `frontend/src/api/client.ts` 拦截器中自动从 `localStorage.auth_token` 读取。

**默认超级管理员**（首次启动自动创建，见 `backend/.env`）：

| 字段 | 默认值 |
|------|--------|
| username | `superadmin` |
| password | `superadmin123` |
| role | `superadmin` |

### 3.4 命名规范

- **JSON 字段**：统一 **camelCase**（如 `collegeId`、`developmentIndex`）
- **查询参数**：与前端 `CollegeScope` 对齐（见下表）

| 参数 | 类型 | 适用模块 | 说明 |
|------|------|----------|------|
| `collegeId` | `string?` | 学院大屏 | 学院编码（`colleges.code`），如 `big-data-ai`；不传则取库中第一个学院 |
| `academicYear` | `string?` | 前端透传 | 如 `2024-2025`；**后端当前未用于过滤** |
| `semester` | `'1' \| '2'?` | 前端透传 | **后端当前未用于过滤** |
| `moduleId` | `string?` | 高潜花名册 | 维度 ID，见 [§7.4](#74-高潜维度-moduleid) |
| `warning_type` | `string` | 预警详情/花名册 | `academic` / `psychological` / `employment` / `credit` |

### 3.5 学院学生范围规则

`collegeId` 解析逻辑（`Utils/DB/read/college_db.py`）：

1. 按 `colleges.code` 精确匹配；
2. 未匹配时 fallback 到 `College.first()`；
3. 学籍范围：`college_id` 外键 **或** `teaching_department` 包含学院名称/简称；
4. 同一 `student_id` 只保留 **最新年级** 一条记录。

## 6. 学院大屏接口

> API 封装：`frontend/src/api/college/`  
> 类型定义：`frontend/src/types/college/api/`

### 6.1 总览

#### `GET /api/v1/college/overview/hub`

学院综合发展指数与 KPI 环。

**查询参数**：`collegeId?`

**响应 `data`**：

```typescript
{
  developmentIndex: number   // 综合发展指数 0-100
  maxScore: number           // 满分，固定 100
  starLevel: number          // 星级 3-5
  kpis: Array<{
    key: 'students' | 'faculty' | 'funding' | 'ranking' | 'satisfaction' | 'influence'
    label: string
    value: number | string
    unit?: string
  }>
}
```

**数据来源**：`student_academic_records` 聚合 + `courses` 计数 + 规则计算（`Utils/Analytics/student_rules.py`）

**前端映射**：`collegeApi.getHub` → `collegeService.fetchOverviewHub` → `CoreHeroGauge`

```typescript
{
  id: string

  status: 'ongoing' | 'completed' | 'delayed' | 'attention' | 'overdue'
```

#### `GET /api/v1/college/tasks/detail`

任务详情页数据。

**响应 `data`**：

```typescript
{
  summary: { total, completed, ongoing, delayed }
  tasks: Array<{
    id, name, progress, status,
    leadDept: string
    deadline: string
    description: string
    milestones: Array<{ label: string; done: boolean }>
  }>
}
```

**数据来源**：`key_tasks`（`scope = 'college'`，可按 `college_id` 过滤）

**前端映射**：

| 接口 | 前端文件 |
|------|----------|
| annual-progress | `collegeApi.getKeyTasks` |
| detail | `collegeDetailApi.getKeyTasksDetail` |

### 6.6 预警与风险

#### `GET /api/v1/college/warnings/overview`

**响应 `data`**：

```typescript
{
  categories: Array<{
    type: 'academic' | 'psychological' | 'employment' | 'credit'
    label: string
    count: number
    momChange: number       // 环比，当前固定 0
  }>
  trend: {
    months: string[]
    series: Array<{ name: string; data: number[] }>
  }
  creditCompletion: {
    threshold: number
    categories: string[]
    junior: number[]
    senior: number[]
  }
}
```

#### `GET /api/v1/college/warnings/{warning_type}`

预警详情列表。

**路径参数 `warning_type`**：`academic` | `psychological` | `employment` | `credit`

**响应 `data`**：

```typescript
{
  type: string
  label: string
  records: Array<{
    name, studentId, major, grade, reason, level
  }>
}
```

#### `GET /api/v1/college/warnings/{warning_type}/roster`

预警花名册（含学生画像字段）。

**响应 `data`**：`{ total, students: RosterStudent[] }`

**预警规则**（`student_rules.build_academic_warnings`）：

| type | 触发条件（示例） |
|------|------------------|
| `academic` | GPA < 2.5；缺考 ≥ 2；补考 ≥ 2 |
| `credit` | 不及格学分 > 0 |
| `employment` | 毕业年级 + GPA < 2.0 |
| `psychological` | 当前无规则，计数为 0 |

**预警等级 `level`**：`high` | `medium` | `low`

### 6.7 高潜学生

#### `GET /api/v1/college/high-potential/overview`

**响应 `data`**：

```typescript
{
  summary: {
    total: number
    change: string           // 如 "+209人"
    coverage: string         // 如 "76.0%"
    activeRate: string
    trend: { months: string[]; counts: number[] }
    kpis: Array<{ label, value, unit? }>
  }
  modules: Array<{
    id: HighPotentialModuleId
    title: string
    desc: string
    cardMetric: { label, value, unit? }
    tags?: string[]
    stats?: Array<{ label, value, unit? }>
  }>
}
```

#### `GET /api/v1/college/high-potential/roster`

**查询参数**：`collegeId?`、`moduleId?`

**响应 `data`**：`{ total, students: RosterStudent[] }`

#### 6.7.1 高潜维度 `moduleId`

| moduleId | 标题 | 判定逻辑摘要 |
|----------|------|--------------|
| `academic` | 学业高潜 | GPA ≥ 3.0 且无不及格；或 GPA ≥ 3.5；或英语优秀 |
| `competition` | 竞赛高潜 | `competition_award_count > 0` |
| `leadership` | 干部奉献高潜 | 按维度计数（当前与竞赛/学业规则部分重叠） |
| `rural` | 双百工程高潜 | 按维度计数 |
| `internship` | 实习项目高潜 | GPA ≥ 3.2 且竞赛 ≥ 1 |
| `career` | 就业高潜 | 学分完成优秀等 |

#### 6.7.2 花名册 `RosterStudent` 结构

```typescript
{
  id: string
  name: string
  gender: string
  studentId: string
  className: string
  major: string
  grade: string              // 如 "2023级"
  counselor: string
  dorm: string               // 当前为空
  gpa: number
  political: string          // 当前为空
  phone: string              // 当前为空（刻意不返回敏感信息）
  hp: string[]               // 高潜维度 ID 列表
  warnings: string[]         // 预警类型列表
  highlight: string
  warnReason?: string
  warnLevel?: string         // 红色预警 | 黄色预警 | 蓝色预警
}
```

> **性能提示**：高潜花名册可能返回数千条记录，前端弹窗加载时注意体验；后续可加分页。

## 9. 超级管理员接口

> 前缀：`/api/v1/admin`  
> **鉴权**：`Authorization: Bearer <token>`，且 `role = superadmin`

### 9.1 可管理资源

`GET /api/v1/admin/resources`

**响应 `data`**：字符串数组，当前支持：

```
accounts
colleges
majors
classes
student_academic_records
courses
key_tasks
school_events
```

### 9.2 通用 CRUD

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/admin/{resource}` | 分页列表 |
| GET | `/admin/{resource}/{item_id}` | 单条详情 |
| POST | `/admin/{resource}` | 创建 |
| PUT | `/admin/{resource}/{item_id}` | 更新 |
| DELETE | `/admin/{resource}/{item_id}` | 删除 |

**列表查询参数**：

| 参数 | 默认 | 说明 |
|------|------|------|
| `page` | 1 | 页码，≥ 1 |
| `page_size` | 20 | 每页条数，1-200 |

**列表响应 `data`**：

```json
{
  "items": [ { } ],
  "total": 100,
  "page": 1,
  "page_size": 20
}
```

**删除响应 `data`**：`{ "deleted": true }`

### 9.3 特殊字段说明

- **`accounts` 创建**：请求体需包含 `password`（明文），后端存 `password_hash`，响应不含密码字段
- **外键字段**：响应中使用 `{field}_id` 形式，如 `college_id`
- **日期时间**：ISO 8601 字符串

### 9.4 主数据表字段摘要

#### `colleges`

| 字段 | 说明 |
|------|------|
| `code` | 学院编码，唯一，前端 `collegeId` 对应此字段 |
| `name` | 学院名称 |
| `short_name` | 简称 |

#### `key_tasks`

| 字段 | 说明 |
|------|------|
| `scope` | `college` / `university` |
| `college_id` | 学院级任务关联学院 |
| `status` | `ongoing` / `completed` / `delayed` / `attention` / `overdue` |
| `milestones` | JSON 数组 `[{label, done}]` |

#### `student_academic_records`

学籍主表，学院大屏核心数据源。关键字段：`student_id`、`grade`、`average_credit_gpa`、`failed_total_credits`、`cet4_score`、`cet6_score`、`competition_award_count`、`major_name`、`teaching_department`、`college_id`。

## 11. 前端架构与对接流程

### 11.1 请求链路

```
View 组件
  → Service（createService：Mock / 真实切换）
    → API 模块（collegeApi / collegeDetailApi）
      → Axios client（baseURL + Token 拦截器）
        → Vite Proxy → FastAPI
          → Router → Service → Utils/DB
```

### 11.2 Adapter 层

后端 DTO 与 UI 展示模型可能不同，Adapter 负责转换：

| 目录 | 作用 |
|------|------|
| `frontend/src/api/college/adapters/` | 学院总览面板 |
| `frontend/src/api/college/adapters/details.ts` | 学院详情页 |
| `frontend/src/types/college/api/` | 后端 DTO 类型 |
| `frontend/src/types/college/view/` | 前端 ViewModel 类型 |

### 11.3 Mock 切换

`frontend/src/api/createService.ts`：

```typescript
const useMock = import.meta.env.VITE_USE_MOCK === 'true'
```

联调时务必设置 `VITE_USE_MOCK=false`，修改 `.env` 后需 **重启** `npm run dev`。

### 11.4 学院大屏加载策略

`CollegeOverviewView` 并发请求 7 个接口（`Promise.allSettled`），互不影响。单接口失败时：

- 控制台输出 `[college] xxx 加载失败`
- 核心指标（hub）全部失败时展示错误页与重试按钮
- 部分失败时其余面板正常展示

# 数据库「小表」设计说明

## 原则

- **字典小表**：配置类信息（奖学金类型、预警类别、高潜标签等）独立成表，增删类型不必改业务表结构。
- **事实小表**：学生获得记录与字典通过外键关联，一行一事，便于查询与扩展字段。
- **避免宽表堆字段**：不把「国家奖学金 / 校一等奖」等写成学籍主表布尔列。

## 已落地表示例

| 表名 | 类型 | 用途 |
|------|------|------|
| `dict_scholarship_types` | 字典 | 奖学金类型（国家级/校级等） |
| `student_scholarships` | 事实 | 学生获奖学金记录 |
| `dict_warning_categories` | 字典 | 预警类别 |
| `dict_high_potential_tags` | 字典 | 高潜标签 |

对应 ORM：`backend/Utils/DB/Models/dict_and_scholarship_models.py`  
初始化 SQL：`docs/sql/small_tables_dict_scholarship.sql`

## 扩展方式

新增一种奖学金（如「企业冠名奖学金」）只需：

1. 在 `dict_scholarship_types` 插入一行；
2. 在 `student_scholarships` 写入学生记录。

无需改 `student_academic_records` 宽表结构。

## 驾驶舱读取

`StudentService.get_dashboard` 从 `student_scholarships` 联表字典名称，填入 dashboard 的 `scholarships: [{ name, year }]`；无记录时保持空数组（前端 mock 仍可演示）。
